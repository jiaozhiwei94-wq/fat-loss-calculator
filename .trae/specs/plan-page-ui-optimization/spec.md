# 减脂计算器计划页 UI/UX 优化 - 产品需求文档

## Overview
- **Summary**: 优化减脂计算器计划页的 UI/UX，通过突出关键信息、分块归纳信息、采用折叠面板设计等方式，提高页面的可读性和用户体验。
- **Purpose**: 解决当前页面信息密度过大，用户难以抓住重点的问题，使页面更加清晰、直观，提高用户的使用体验。
- **Target Users**: 使用减脂计算器的用户，特别是需要查看详细饮食和训练计划的用户。

## Goals
- 突出页面顶部关键信息，使“每日总热量目标”和“预计达成目标时间”更加醒目
- 分块归纳信息，使页面结构更加清晰
- 采用折叠面板设计，减少初始页面的信息密度
- 保留核心数据可见，确保用户在卡片展开前仍能看到最重要的信息

## Non-Goals (Out of Scope)
- 不改变现有的数据计算逻辑
- 不修改页面的整体设计风格
- 不添加新的功能模块

## Background & Context
- 当前计划页包含饮食计划和训练计划两个主要部分
- 每个部分都包含多个详细信息模块，导致页面信息密度过大
- 用户需要滚动很长的页面才能查看所有信息，难以抓住重点

## Functional Requirements
- **FR-1**: 突出显示关键信息，将“每日总热量目标”和“预计达成目标时间”显示在页面顶部
- **FR-2**: 实现折叠面板组件，用于分组和收起详细信息
- **FR-3**: 确保核心数据在卡片展开前可见
- **FR-4**: 优化页面布局，提高信息的可读性

## Non-Functional Requirements
- **NFR-1**: 页面加载速度应保持不变或更快
- **NFR-2**: 折叠面板的展开/收起动画应流畅
- **NFR-3**: 页面应在不同设备上都能正常显示
- **NFR-4**: 保持与现有设计风格的一致性

## Constraints
- **Technical**: React TypeScript 项目，使用现有的 UI 库和组件
- **Business**: 不添加新的依赖
- **Dependencies**: 现有的 React 组件和样式系统

## Assumptions
- 用户希望快速获取关键信息，而不是被过多的详细信息所淹没
- 折叠面板设计可以有效减少页面的信息密度
- 核心数据应该在卡片展开前可见，以便用户快速了解重要信息

## Acceptance Criteria

### AC-1: 关键信息突出显示
- **Given**: 用户访问计划页
- **When**: 页面加载完成
- **Then**: “每日总热量目标”和“预计达成目标时间”应该以醒目的方式显示在页面顶部
- **Verification**: `human-judgment`

### AC-2: 折叠面板功能
- **Given**: 用户访问计划页
- **When**: 页面加载完成
- **Then**: 详细信息应该默认收起，只显示核心数据
- **Verification**: `programmatic`

### AC-3: 核心数据可见
- **Given**: 用户访问计划页
- **When**: 页面加载完成，折叠面板处于收起状态
- **Then**: 用户应该能看到饮食计划的“每日三大营养素分配”和训练计划的“训练类型”和“建议时长”
- **Verification**: `human-judgment`

### AC-4: 折叠面板交互
- **Given**: 用户点击折叠面板的标题
- **When**: 点击事件触发
- **Then**: 折叠面板应该平滑展开或收起
- **Verification**: `human-judgment`

### AC-5: 响应式设计
- **Given**: 用户在不同设备上访问计划页
- **When**: 页面加载完成
- **Then**: 页面应该在不同设备上都能正常显示，布局应该适应不同的屏幕尺寸
- **Verification**: `human-judgment`
