# 减脂计算器结果页 UI/UX 优化 - 产品需求文档

## Overview
- **Summary**: 优化减脂计算器结果页的 UI/UX，通过重新组织页面结构、引入新页面和优化视觉设计，提高页面的可读性和用户体验。
- **Purpose**: 解决当前页面信息密度过大、缺乏明确视觉重点和用户引导的问题，使页面更加清晰、直观，提高用户体验。
- **Target Users**: 使用减脂计算器的用户，特别是需要查看详细饮食和训练计划的用户。

## Goals
- 突出页面顶部的关键信息，使“每日总热量目标”和“预计达成目标时间”更加醒目
- 优化“饮食计划”和“训练计划”模块，减少主页面的信息密度
- 为详细内容创建新的页面，通过导航按钮链接
- 提高页面的视觉层次和区分度

## Non-Goals (Out of Scope)
- 不改变现有的数据计算逻辑
- 不修改页面的整体设计风格
- 不添加新的功能模块

## Background & Context
- 当前计划页包含饮食计划和训练计划两个主要部分
- 每个部分都包含多个详细信息模块，导致页面信息密度过大
- 用户需要滚动很长的页面才能查看所有信息，难以抓住重点
- 折叠面板虽然减少了部分信息密度，但仍不够简洁

## Functional Requirements
- **FR-1**: 优化页面顶部总览区域，突出显示“每日总热量目标”和“预计达成目标时间”
- **FR-2**: 为“饮食详情”和“训练详情”创建新的页面组件
- **FR-3**: 在主计划页上添加导航按钮，链接到新的详情页面
- **FR-4**: 优化“饮食计划”和“训练计划”模块，只显示核心信息

## Non-Functional Requirements
- **NFR-1**: 页面加载速度应保持不变或更快
- **NFR-2**: 页面导航应流畅，动画效果应自然
- **NFR-3**: 页面应在不同设备上都能正常显示
- **NFR-4**: 保持与现有设计风格的一致性

## Constraints
- **Technical**: React TypeScript 项目，使用现有的 UI 库和组件
- **Business**: 不添加新的依赖
- **Dependencies**: 现有的 React 组件和样式系统

## Assumptions
- 用户希望快速获取关键信息，而不是被过多的详细信息所淹没
- 详细信息可以通过导航到新页面来查看
- 视觉层次和区分度可以通过颜色、大小和布局来实现

## Acceptance Criteria

### AC-1: 页面顶部总览区域
- **Given**: 用户访问计划页
- **When**: 页面加载完成
- **Then**: “每日总热量目标”和“预计达成目标时间”应该以醒目的方式显示在页面顶部，其中“每日总热量目标”是页面上最大、最突出的数字
- **Verification**: `human-judgment`

### AC-2: 饮食计划模块
- **Given**: 用户访问计划页
- **When**: 页面加载完成
- **Then**: “饮食计划”模块应该只显示核心的“每日三大营养素分配”数据，详细内容应通过“查看详细饮食建议”按钮链接到新页面
- **Verification**: `human-judgment`

### AC-3: 训练计划模块
- **Given**: 用户访问计划页
- **When**: 页面加载完成
- **Then**: “训练计划”模块应该只显示核心的“训练建议”和“预计运动消耗”，详细内容应通过“查看详细训练指导”按钮链接到新页面
- **Verification**: `human-judgment`

### AC-4: 新页面导航
- **Given**: 用户点击“查看详细饮食建议”或“查看详细训练指导”按钮
- **When**: 点击事件触发
- **Then**: 用户应该被导航到相应的详情页面，页面内容应该完整显示
- **Verification**: `programmatic`

### AC-5: 响应式设计
- **Given**: 用户在不同设备上访问计划页和详情页面
- **When**: 页面加载完成
- **Then**: 页面应该在不同设备上都能正常显示，布局应该适应不同的屏幕尺寸
- **Verification**: `human-judgment`
